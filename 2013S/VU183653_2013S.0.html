What's this course about?

  • Learning how to apply functional programming techniques for building industrial-grade systems.
  • Learning what facilities Haskell offers over traditionally deployed languages for building robust software.
  • Learning about GHC internals in order to be able to reason about performance and resource usage of Haskell programs.
  • Applying some of the concepts shown in this course in a project of your own choice.

Why Haskell?

  • Small language core
  • Expressive power ( reduces accidental complexity)
  • Lazy evaluation ( infinite data structures, composability, EDSLs, extensions as libraries)
  • Advanced static type-system ( correctness)
  • Purely functional ( correctness)
  • Immutable data ( correctness)
  • Built-in support for various models of concurrency and parallelism
  • Can be compiled ( performance) as well as interpreted ( speedy development)

Haskell's popularity

Case Study: BB Ticketing

BB's new online & mobile ticketing system for the austrian railway

Case Study: BB Ticketing (cont.)

  • Collection of components talking to each other via web-service calls (SOA)
  • Some of the backend components are implemented in Haskell, most notably the central offer/fare calculation engine:
    • Given a set of objects ("it's people!") to be conveyed and
    • a list of connections (as provided by the time-table service),
    • this component computes optimal offers in accordance with the product definition database
    • while taking into account dynamic information such as seat reservation and contingent availability.
  • One of the most complex components and yet no major bugs known since go-live last year

Your own project!

  • Consists of a Haskell-related project of your choice
    • Form project team of 1-3 people and agree on a project idea
    • You may reuse project from a different class
    • Submit your project proposal ASAP
  • Don't hesistate to ask for help/guidance if you're stuck on a Haskell problem
  • Getting a grade requires your team to present your work at the end of this course
    • Self-evaluate your project experience (the good things as well as the problems that you encountered).
    • Compare to traditional implementation techniques (e.g. Java).
    • Submit a short paper (will be distributed to other students)
    • ...and finally give a short presentation (about 15 minutes) and be prepared to respond to questions from the audience afterwards.
